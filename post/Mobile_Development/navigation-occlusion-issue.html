# menu bar 滿版時 navigation 遮擋問題 - 2025 新解法

# tags: web, ios, android, safari, chrome, viewport

# 快速結論

以前要用 JS 監聽 resize 很麻煩，現在有新的 CSS viewport 單位可以直接用！

.menu-container {
  @media (max-width: 1279px) {
    height: 100svh;  /* 就這樣，不用 JS！ */
  }
}

# 新的 Viewport 單位（2022 年推出）

有三個新單位可以用：

單位說明什麼時候用
svh最小可視高度（導航欄都顯示時）✅ 推薦，最穩定
lvh最大可視高度（導航欄隱藏時）想要沉浸式體驗
dvh動態調整（跟著導航欄變化）⚠️ 會跳來跳去

# 實際範例

# 推薦用法：svh

.menu-container {
  @media (max-width: 1279px) {
    height: 100svh;
  }
}
1
2
3
4
5

好處：

  • 不用寫 JavaScript
  • iOS 和 Android 都支援
  • 不會被導航欄擋住
  • 滾動時不會有白底
  • 效能好

# 如果需要動態調整：dvh

.menu-container {
  @media (max-width: 1279px) {
    height: 100dvh;  /* 會隨著導航欄顯示/隱藏調整 */
  }
}

但這個會跳來跳去，通常不推薦。

# 保險起見加 fallback

.menu-container {
  @media (max-width: 1279px) {
    height: 100vh;      /* 舊瀏覽器 */
    height: 100svh;     /* 新瀏覽器 */
  }
}

# 怎麼選單位？

情況用哪個為什麼
滿版選單svh最穩，不會被擋
Hero Sectiondvh動態調整比較漂亮
Modal / Dialogsvh確保看得到
Sticky Footersvh不會被底部導航擋住

大部分情況用 svh 就對了。

# 注意事項

  1. 記得加 fallback

    height: 100vh;    /* 舊瀏覽器 */
height: 100svh;   /* 新瀏覽器 */
    1
    2

  2. dvh 會跳動
    如果元素高度很重要，別用 dvh，會一直變來變去

  3. iOS 瀏海要處理
    記得加 env(safe-area-inset-bottom)

  4. 實機測試
    模擬器有時候行為不一樣，要用真機測

# 總結

# 2025 年就這樣寫：

.menu-container {
  height: 100vh;    /* fallback */
  height: 100svh;   /* 主要用這個 */
}
